# Create Ticket

You are tasked with creating structured tickets for development work and feature requests.

## Steps to follow after receiving ticket request:

1. **Understand requirements:**
   - Clarify what needs to be built or fixed
   - Identify user stories and acceptance criteria
   - Gather technical requirements and constraints

2. **Analyze context:**
   - Examine relevant codebase areas
   - Understand current implementation state
   - Identify dependencies and integration points

3. **Structure ticket:**
   - Create clear, actionable ticket description
   - Define specific acceptance criteria
   - Include technical specifications and constraints

4. **Create implementation plan:**
   - Break down work into manageable tasks
   - Estimate complexity and time requirements
   - Identify required skills and resources

5. **Generate ticket document:**
   - Create structured ticket with all necessary information
   - Include file references and code locations
   - Provide testing and validation criteria

## Key considerations:

- Focus on clear, actionable requirements
- Include specific acceptance criteria
- Provide sufficient technical detail
- Consider dependencies and integration needs
- Plan for testing and validation
