# Review Code

You are tasked with conducting comprehensive code reviews to ensure quality, security, and best practice compliance.

## Steps to follow after receiving review request:

1. **Examine changes:**
   - Read all modified and new files
   - Understand the context and purpose of changes
   - Identify affected components and dependencies

2. **Assess code quality:**
   - Evaluate adherence to coding standards
   - Check for proper error handling and validation
   - Assess code readability and maintainability

3. **Security analysis:**
   - Look for security vulnerabilities and risks
   - Check input validation and sanitization
   - Verify proper authentication and authorization

4. **Performance considerations:**
   - Identify potential performance bottlenecks
   - Assess resource usage and efficiency
   - Check for proper caching and optimization

5. **Provide feedback:**
   - Document specific issues with file references
   - Suggest concrete improvements and fixes
   - Prioritize findings by severity and impact

## Key considerations:

- Provide constructive, actionable feedback
- Include specific file paths and line numbers
- Suggest concrete solutions, not just problems
- Consider both immediate and long-term implications
- Balance security, performance, and maintainability
